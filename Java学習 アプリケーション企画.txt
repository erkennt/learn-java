## ここまで


◆カードゲーム
・概要
  作成するアプリケーションは、多人数参加型のブラウザゲームです。
  ユーザーの登録を行うと、一定の資産がユーザーに割り当てられ、ユーザーは資産を賭けてカードを引きます。
  用意されたカードゲームの条件に合うカードを引けば資産は増え、合わない場合は減ります。
  同じユーザーが連続でカードを引く事はできず、別のユーザーがカードを引くまで待つ必要があります。
  一度カードを引くと、一定時間カードを引くことができません。
   カードは0〜9までを扱います。

・カードゲームの種類とルール
  [共通ルール]
    賭ける金額の桁数を選んでカードを引きます(桁数は所持する資産に準ずる)
    成功するごとに10進数で倍率が積みあがり、失敗すると倍率は1に戻ります。
    カードを引いた結果によって積みあがった倍率×桁数の金額が総資産から増減します。

  [ゲームの種類]
    -ドーナツ
      前のカードと同じ数を出さないようにするゲーム。
      前のカードと異なるカードを出すと成功、同じ数の場合は失敗。

    -High&Low
      次のカードが前のカードより大きいか小さいかを当てるゲーム。
      どちらかを選んで、選んだ通りの結果なら成功、異なれば失敗。


・作成機能
  -一般ユーザー登録
    ユーザー情報入力(ニックネーム、id、パスワード、メールアドレス、生年月日等)
    メール認証


  -ログイン / ログアウト（一般ユーザー）
   ユーザー情報の表示
   ユーザー情報編集
   パスワードの変更
   ゲームへの参加


  -ログイン / ログアウト（管理者）
   ユーザー情報の操作
     ユーザー一覧
     ユーザー検索
     ユーザー情報 / ユーザーのゲーム情報編集
     ユーザー停止 / 復旧


  -ドーナツゲーム
   参加ユーザーが過去に引いたカードの表示
   カードを引いたユーザーと金額の増減のログを表示
   残りの待機時間の動的な表示
   ゲーム実行


  -High&Low
   ゲームロジック以外はドーナツゲームと同様


  -非ログイン機能
   ゲーム状況の表示 
   ID / パスワード再送


・作成予定機能
  -ログイン / ログアウト（管理者）
   ゲーム情報の操作
     ゲーム設定の変更
     ゲーム進行状態のリセット
     ゲームログの削除
     全ユーザーの待機時間リセット
     全ユーザーの所持金を初期値に戻す

   ゲームデータの操作
     全データ削除 (登録者、資産、カードのログ等)
     全データをCSVファイルへ出力
     全データをCSVでデータの一括登録
